Totally Tanktical

"Submachines: Totally Tanktical" is a topdown, tankbuilding, rts-like game, where you construct a tank from modular parts to combat other tanks made of parts, in a post-apocalyptic world where mankind has reestablished civilization under the sea.

I was brought onto this student project and made to be the lead programmer.

Programming-wise, our programming team inherited the project in a prototype state and were tasked with adding to it. This involved remaking many important system that had convoluted algorithms, confusing architecture, compounded by a lack of proper documentation. As the project progressed, we managed to remake or replace these systems and made them simpler and easier to work with, as well as improve the overall quality of the systems in the game.

My responsibilities and priorities were:

  • Communicate with the product owners to determine what features they wanted in the game
  • Ensure that the product owners were kept up to date on all programming tasks, as well as describe the technical problems in a way they could understand
  • Delegate tasks to programmers to ensure each member knew what each was doing without stepping on each other's toes and establishing priorities of what needed to get finished before moving on to other things.
  • Keep up to date with every member of the programming team and the work they were doing, giving feedback and having peer discussions about their work.
  • Create tools and libraries that other members of the team could use, such as the designers, artists, and other programmers.
  • Manage a PlasticSCM repo for our version control, ensuring that people were using it properly so that everyone could work on the project simultaneously
  • Assemble the builds for the game, rallying other members to get their changes in on time, and making bug fixes and adjustments as needed for the game
  • Showing off the game at expo events, like PAX East and the Clark IGDA expo.

Systems I worked on:

  • Block pre-placement visual: There was no visual before this system was implemented. It required carefully navigating the existing implementation, and a few hacky lines that were necessary due to the project relying heavily on OOP and inheritance. It revealed some concerning performance heavy algorithms related to placing blocks that I later documented.
  • Automatic object initializer: Using Unity's Editor namespace, I created a way to initialize necessary objects for scenes, as opposed to needing the game designers to put it in manually. The tool would load in a prefab to the scene when it was loaded in the editor, and if it was in the list of scenes that needed to have objects loaded, it would load them.

Systems I remade:

  • Camera movement: I rebuilt the camera system that was overly complicated and difficult to work with, into a slimmed down system with the same functionality. It has the ability to zoom in and out, pan away from the tank and snap back to it, and the ability to rotate around the object of interest. The previous system relied heavily on raycasts, which was buggy and difficult to understand.
  • Material Assignment: I rebuilt the systems that were responsible for assigning the right textures for the blocks when being attached and detached from tanks. The old system was difficult to work with, as the previous coder had introduced multiple steps to reassigning materials, and it was a constant source of trouble. I later expanded the systems to handle assigning proper textures for the different damaged states of blocks. It was important to me to make it easy for others to add new kinds of blocks to the system, as well as make it easy to assign the proper materials.

DELETE SYSTEM 32

Play It Now! View game on itch.io

"DELETE SYSTEM 32" is a sci-fi, shmup game where you play as a rogue computer virus attempting to destroy a computer's operating system. You are thwarted by the system's antivirus software, and must destroy it to achieve your goal. Built in Unity as part of IGDA Clark's 48 hr 2024 February Frenzy Game Jam

I was the sole programmer for this game.

Game features:

  • A finite state machine to cycle different enemy AI behaviors.
  • Procedurally generating the towers and enemy spawners in the playspace.
  • Making the screen endlessly scroll.
  • A system for powerups.
  • Cardinal movement system, and the ability to freely rotate to aim.
  • Modular UI components for non-programmers to build UI with.

Ship Shoot

View game on itch.io

"Ship Shoot" is a sci-fi, arcade, shmup game where you are a traveling combat ship. You are confronted by waves of enemy ships that attempt to shoot you down.

This was my first game, I made it in two weeks during the "My First Game Jam" jam on Itch.io. It was my first time trying to create a playable game using Scratch. Every asset and feature was made by me.

It taught me many of the finer details of how video games work from an almost no engine baseline, and it's a reminder of how far I've come, and how much I understood even without much background in programming at the time.

It features:

  • Custom hit detection
  • Custom entity tracking system
  • Custom projectile object system
  • Custom Enemy AI
  • Custom Text Printing
